Design system · v1.0

A token-driven, accessible UI kit

One CSS-variable token layer drives light and dark. Built with Tailwind CSS and the Lucide icon library. Every pattern is held to the accessibility and anti-AI-pattern guidelines below.

Token-driven

Colors, radii, type as variables — themeable with zero duplication.

Accessible

WCAG AA+ contrast, visible focus, keyboard-first, reduced-motion.

De-slopped

Flat surfaces, one icon family, a chosen palette — no AI tells.

Color

Brand is #E61876 (magenta). Neutrals are biased cool; semantic colors are a separate, reserved system.

Brand ramp

Neutrals

Semantic — reserved; always paired with an icon + label

Gradient

The brand gradient (--brand-gradient) runs #E61876 → #94104B. Use it for hero/marketing surfaces only; keep flat --brand for interactive UI so contrast stays predictable.

Ship accessible UI, faster.
background: linear-gradient(135deg, #E61876 0%, #94104B 100%);

8 gradient variations are available in the brand-gradient chooser; this is option 01 (Diagonal).

Typography

A native system-ui sans stack (Regular 400 / Medium 500 / Bold 700) — swap in your own brand typeface. UI labels and data use a monospace stack.

Elevation

Two levels only. Most surfaces are flat (border-only); a single shadow-lift marks what genuinely floats (menus, popovers, dialogs).

Icons

One library: Lucide (2px stroke, currentColor). Never emoji. Sizes 16 / 20 / 24. Decorative → aria-hidden; icon-only control → aria-label.

Buttons

Five style variants. 44px min touch target, visible focus ring, semantic red is separate from brand.

Forms

Every field has a label; errors pair an icon with text (never color alone) and set aria-invalid.
